U3O1·KK01 Unit 3 – Outcome 1 — Programming ▲ Very likely
Emerging trends in programming using AI
2026 signal: AI ethics & emerging trends named as a leading 2026 emphasis.
Q01 Identify Section A · Multiple choice 1 mark
A developer uses an AI coding assistant to generate a function. The MOST significant reliability concern is that the generated code may
A. take longer to execute than hand-written code.
B. contain logic errors or insecure patterns the developer does not detect.
C. use a different naming convention to the rest of the project.
D. require an internet connection to run.
Indicative answer / marking noteB. AI-generated code can embed subtle logic/security flaws; over-trust without review is the key reliability risk (A/C/D are minor or not reliability).
Q02 Define Section B · Short answer 1 mark
Define the term ‘large language model’ in the context of AI-assisted programming.
Indicative answer / marking noteA model trained on large volumes of text/code that predicts likely next tokens to generate or complete code and explanations.
Q03 Outline Section B · Short answer 2 marks
Outline two ways AI tools are changing how programmers develop software solutions.
Indicative answer / marking noteAny 2 × 1: code generation/autocompletion; automated debugging or test generation; natural-language to code; rapid prototyping; documentation generation.
Q04 Describe Section B · Short answer 2 marks
Describe one benefit and one risk of using an AI assistant to generate code for a commercial solution.
Indicative answer / marking noteBenefit (1): speed/productivity, boilerplate reduction, learning support. Risk (1): incorrect/insecure code, IP/licensing issues, over-reliance reducing developer understanding.
Q05 Explain Section B · Short answer 3 marks
A student submits AI-generated code without testing it. Explain why this is poor development practice, referring to reliability and accountability.
Indicative answer / marking noteReliability (1–2): AI output is probabilistic and may be wrong/insecure; untested code may fail on edge cases. Accountability (1): the developer remains responsible for correctness; lack of testing breaches professional practice.
Q06 Discuss Section B · Short answer 4 marks
Discuss the ethical issues that arise when a developer uses AI to generate code that is then sold to a client.
Indicative answer / marking noteIssues (any 4, 1 each): ownership/IP of generated code; potential plagiarism of training data; disclosure/transparency to the client; liability for defects; bias in output; data privacy of prompts. Reward balanced discussion of more than one viewpoint.
Q07 Identify Section C · Case study 2 marks
An organisation plans to embed an AI feature that recommends products to customers. Identify and describe one bias-related risk of this feature.
Indicative answer / marking noteIdentify (1): algorithmic/training-data bias. Describe (1): model may systematically disadvantage a group (e.g. recommend poorly for under-represented customers) because training data was unrepresentative.
Q08 Explain Section C · Case study 3 marks
Explain how a development team could responsibly integrate an AI coding assistant into their workflow while managing reliability concerns.
Indicative answer / marking noteAny 3 × 1: human code review of all AI output; comprehensive testing (unit/integration); use AI for drafts not final logic; verify against requirements; document AI use; security scanning.
Q09 Compare Section C · Case study 4 marks
Compare AI-assisted development with traditional hand-coding in terms of speed, reliability, and developer skill.
Indicative answer / marking noteSpeed (1): AI faster for boilerplate. Reliability (1–2): hand-coding more predictable; AI needs verification. Skill (1): AI may erode/accelerate skill depending on use. Must explicitly compare, not list.
Q10 Justify Section C · Case study 4 marks
Ming asks an AI assistant to generate code for a payment module. Justify whether the organisation should deploy this code as-is.
Indicative answer / marking noteJustification should conclude NO (or conditional): payment = high risk/security-sensitive; AI output unverified, possible vulnerabilities, compliance/PCI obligations, liability. Require review, testing, and audit before deployment. Reward a clear position supported by reasons.
U3O1·KK03 Unit 3 – Outcome 1 — Programming ▲ Very likely
Design tools for representing modules — IPO charts, pseudocode, object descriptions
2026 signal: Pseudocode is ~10–15% of every paper — effectively guaranteed.
Q01 Identify Section A · Multiple choice 1 mark
In an IPO chart, a running total that is updated each loop iteration is best classified as
A. an input.
B. a process.
C. an output.
D. a constant.
Indicative answer / marking noteB. Accumulating a total is processing logic (input = raw values; output = displayed result).
Q02 State Section B · Short answer 1 mark
State the purpose of pseudocode in the design stage of software development.
Indicative answer / marking noteTo represent the logic/steps of an algorithm in a language-independent, human-readable way before coding.
Q03 Describe Section B · Short answer 2 marks
Describe what an IPO chart shows and one situation where it is useful.
Indicative answer / marking noteDescribe (1): maps the Inputs, Processing and Outputs of a module/solution. Use (1): clarifying data flow / requirements before designing the algorithm.
Q04 Explain Section B · Short answer 2 marks
Write pseudocode that reads 10 integers and outputs the largest value.
Indicative answer / marking noteInitialise largest to first/very small; loop 10 times reading n; IF n > largest THEN largest ← n; output largest. Reward correct initialisation, loop, comparison, output.
Q05 Outline Section B · Short answer 3 marks
Trace the following pseudocode and state the final value of total. BEGIN total←0; FOR i←1 TO 4 DO total←total+i END FOR; PRINT total END
Indicative answer / marking notetotal = 1+2+3+4 = 10. (Award method marks for correct iteration values 1,3,6,10.)
Q06 Describe Section B · Short answer 2 marks
Describe the components you would include in an object description for a button object.
Indicative answer / marking noteObject name/type; properties (e.g. caption, enabled, position); methods/events (e.g. onClick); behaviour/purpose. Any 2 components with detail.
Q07 Explain Section C · Case study 3 marks
Write pseudocode for an algorithm that validates a password is at least 8 characters and contains a digit, looping until valid.
Indicative answer / marking noteREPEAT input pwd; valid ← (length(pwd) ≥ 8) AND containsDigit(pwd); IF NOT valid THEN message UNTIL valid. Reward loop, both conditions, exit logic.
Q08 Describe Section C · Case study 2 marks
Describe two advantages of producing pseudocode before writing program code.
Indicative answer / marking noteAny 2 × 1: language-independent logic check; easier for others to follow/review; reduces coding errors; supports planning/communication.
Q09 Explain Section C · Case study 3 marks
A team's pseudocode uses an inclusive boundary (≤) where the requirement specifies ‘under 18’. Explain the impact of this error and how a design tool helps catch it.
Indicative answer / marking noteImpact (1–2): off-by-one/boundary error includes 18-year-olds incorrectly — wrong results. Design tool (1): reviewing pseudocode against requirements (or desk-checking/trace) exposes the boundary before coding.
Q10 Compare Section C · Case study 4 marks
Compare pseudocode and an object description as design tools, including what each best represents.
Indicative answer / marking notePseudocode (1–2): represents procedural logic/algorithms; sequence/selection/iteration. Object description (1–2): represents an object's properties, methods and behaviour. Must compare scope/purpose, not just define each.
U3O1·KK06 Unit 3 – Outcome 1 — Programming ▲ Very likely
Characteristics of data sources — files, CSV and XML
2026 signal: CSV/XML named for 2026; XML caused 58% zero in 2025.
Q01 Identify Section A · Multiple choice 1 mark
Which statement best describes XML compared with CSV?
A. XML stores values separated by commas with no structure.
B. XML uses nested tags to give data a hierarchical, self-describing structure.
C. XML can only store numeric data.
D. XML files are always smaller than equivalent CSV files.
Indicative answer / marking noteB. XML is hierarchical and self-describing via tags (CSV is flat/delimited; D is false — XML is usually larger).
Q02 Define Section B · Short answer 1 mark
Define the term ‘delimiter’ as used in a CSV file.
Indicative answer / marking noteA character (e.g. a comma) that separates individual data values/fields within a record.
Q03 Describe Section B · Short answer 2 marks
Describe two characteristics of a CSV file.
Indicative answer / marking noteAny 2 × 1: plain text; values separated by a delimiter (comma); one record per line; no inherent data types; flat/non-hierarchical; widely portable.
Q04 Explain Section B · Short answer 2 marks
Explain why XML is described as ‘self-describing’.
Indicative answer / marking noteTags label the meaning of each element (e.g. ), so the structure and meaning of the data travel with the data, without a separate schema being required to interpret field names.
Q05 Compare Section B · Short answer 3 marks
Compare CSV and XML as data sources for transferring structured data between systems.
Indicative answer / marking noteCSV (1): compact, simple, flat, fast to parse, no hierarchy/types. XML (1): hierarchical, self-describing, supports nesting/attributes, larger/verbose. Comparison (1): choose CSV for simple tables, XML for nested/complex data. Must compare.
Q06 Outline Section B · Short answer 2 marks
Write a small XML fragment that represents one student with a name and a year level.
Indicative answer / marking note...11 — reward well-formed nesting and a root/parent element with matching open/close tags.
Q07 Explain Section C · Case study 3 marks
A solution imports a CSV where one row has a missing field. Explain why this can cause an error and how the program should handle it.
Indicative answer / marking noteCause (1–2): field count mismatch shifts/loses values, type conversion fails. Handling (1): validate field count, use existence/type checks, skip/flag malformed rows, error-handle the import.
Q08 Describe Section C · Case study 2 marks
Describe one advantage and one disadvantage of using XML over CSV for storing an organisation's product catalogue.
Indicative answer / marking noteAdvantage (1): supports nested/structured data (categories, variants) and validation. Disadvantage (1): more verbose, larger files, slower/more complex to parse.
Q09 Justify Section C · Case study 3 marks
A developer must exchange hierarchical configuration data between two systems. Justify whether CSV or XML is more appropriate.
Indicative answer / marking noteConclude XML: hierarchical/nested config needs structure CSV cannot represent; self-describing and validatable. Reward clear position + reasons tied to hierarchy.
Q10 Explain Section C · Case study 3 marks
Explain how the choice of data source (CSV vs XML) affects the validation a developer must perform on import.
Indicative answer / marking noteCSV (1–2): must check delimiter integrity, field count, types manually. XML (1–2): can validate well-formedness/schema, but must still check element presence and value ranges. Link choice to validation effort.
U3O1·KK07 Unit 3 – Outcome 1 — Programming ▲ Very likely
Principles of object-oriented programming
2026 signal: OOP = 8 marks, avg only 3.5/8 — flagged the top priority.
Q01 Identify Section A · Multiple choice 1 mark
Hiding an object's internal data and exposing only methods to access it is known as
A. inheritance.
B. polymorphism.
C. encapsulation.
D. abstraction.
Indicative answer / marking noteC. Encapsulation bundles data with methods and restricts direct access.
Q02 Define Section B · Short answer 2 marks
Define the terms ‘class’ and ‘object’ and state the relationship between them.
Indicative answer / marking noteClass (1): a template/blueprint defining attributes and methods. Object (1): a specific instance of a class. Relationship: an object is created from a class.
Q03 Describe Section B · Short answer 2 marks
Describe the principle of inheritance in object-oriented programming.
Indicative answer / marking noteA subclass/child class acquires (inherits) the attributes and methods of a parent/superclass, and may extend or override them, promoting reuse.
Q04 Explain Section B · Short answer 2 marks
Explain one benefit of encapsulation in a large software project.
Indicative answer / marking noteProtects internal state from invalid external changes / reduces coupling, so internal implementation can change without breaking other code; improves maintainability and reliability.
Q05 Describe Section B · Short answer 2 marks
Describe what polymorphism allows a programmer to do, with an example.
Indicative answer / marking noteAllows the same method name/interface to behave differently across classes (e.g. draw() on Circle vs Square). Reward definition + valid example.
Q06 Contrast Section B · Short answer 3 marks
Contrast a class and an object, and contrast an attribute and a method.
Indicative answer / marking noteClass vs object (1–2): template vs instance. Attribute vs method (1): attribute = data/property of object; method = behaviour/operation. Must contrast each pair.
Q07 Outline Section C · Case study 2 marks
An Account class has subclasses SavingsAccount and ChequeAccount. Outline how inheritance is used here.
Indicative answer / marking noteAccount holds shared attributes/methods (balance, deposit); the two subclasses inherit these and add/override specific behaviour (e.g. interest, overdraft).
Q08 Explain Section C · Case study 3 marks
Explain how encapsulation and inheritance together improve the maintainability of an object-oriented solution.
Indicative answer / marking noteEncapsulation (1–2): localises change behind interfaces. Inheritance (1–2): centralises shared code so a fix in the parent propagates. Together reduce duplication and limit ripple effects.
Q09 Justify Section C · Case study 4 marks
A team debates using OOP versus procedural code for a banking application with many account types. Justify which paradigm is more suitable.
Indicative answer / marking noteConclude OOP: many related entities (account types) benefit from classes/inheritance/encapsulation for reuse, data protection, scalability. Reward position + at least two OOP principles applied to the scenario.
Q10 Describe Section C · Case study 2 marks
Describe one disadvantage of using object-oriented programming for a very small, simple script.
Indicative answer / marking noteAdds overhead/complexity (class structure) disproportionate to the task; can be slower to write/understand than a short procedural script.
U3O1·KK14 Unit 3 – Outcome 1 — Programming ▲ Very likely
Debugging and testing techniques
2026 signal: Testing tables explicitly named as a 2026 revisit.
Q01 Identify Section A · Multiple choice 1 mark
A test case that uses a value exactly on the limit of an accepted range is testing
A. typical data.
B. boundary data.
C. invalid data.
D. erroneous logic.
Indicative answer / marking noteB. Boundary (edge) data tests the limits of valid ranges.
Q02 State Section B · Short answer 1 mark
State the three categories of test data commonly used to test a solution.
Indicative answer / marking noteValid/typical (normal), boundary (edge), and invalid/erroneous data.
Q03 Describe Section B · Short answer 2 marks
Describe the columns you would include in a test table (test data plan).
Indicative answer / marking noteTest no./case; input/test data; data type (valid/boundary/invalid); expected result; actual result; pass/fail. Award for naming purposeful columns.
Q04 Explain Section B · Short answer 2 marks
A field accepts ages 18–65. Explain why both 17 and 18 should appear in the test table.
Indicative answer / marking note17 (1): invalid/boundary-below should be rejected. 18 (1): valid boundary should be accepted; together they confirm the boundary is handled correctly (catches off-by-one).
Q05 Outline Section B · Short answer 2 marks
Outline the difference between a syntax error and a logic error.
Indicative answer / marking noteSyntax (1): breaks language rules; code won't compile/run. Logic (1): runs but produces wrong output; harder to detect.
Q06 Describe Section B · Short answer 2 marks
Describe two debugging techniques a programmer could use to locate a logic error.
Indicative answer / marking noteAny 2 × 1: breakpoints/stepping; watch variables; print/trace statements; desk-checking; commenting out code; using a debugger to inspect state.
Q07 Compare Section C · Case study 3 marks
Compare desk-checking and using a debugger as techniques for finding errors.
Indicative answer / marking noteDesk-check (1): manual trace on paper, no execution, good early/conceptual. Debugger (1): runtime inspection, breakpoints, exact state. Comparison (1): desk-check catches logic in design; debugger catches runtime behaviour. Must compare.
Q08 Explain Section C · Case study 3 marks
For a function that calculates a 10% discount on orders over $100, design three test cases (valid, boundary, invalid) and state the expected output of each.
Indicative answer / marking noteE.g. $150 → $15 discount (valid); $100 → $0 / boundary behaviour per spec; -$50 or text → rejected (invalid). Reward one of each category with correct expected result.
Q09 Explain Section C · Case study 3 marks
Explain why testing should use a documented test table rather than ad-hoc testing.
Indicative answer / marking noteAny 3 × 1: repeatable/consistent; coverage of valid+boundary+invalid; evidence for evaluation/accountability; expected vs actual makes failures explicit; supports regression testing.
Q10 Justify Section C · Case study 3 marks
A developer says testing only valid data is sufficient. Justify why this is inadequate.
Indicative answer / marking noteInadequate: valid-only ignores boundary (off-by-one) and invalid input (crashes/security). Robust solutions must reject bad input and handle edges. Reward clear stance + boundary AND invalid reasoning.
U3O2·KK03 Unit 3 – Outcome 2 — Analysis & Design ▲ Very likely
Project management — Gantt charts, dependencies, critical path, milestones
2026 signal: Gantt / critical-path annotations recur; ~40% scored zero in 2025.
Q01 Identify Section A · Multiple choice 1 mark
On a Gantt chart, the longest sequence of dependent tasks that determines the minimum project duration is the
A. milestone.
B. slack time.
C. critical path.
D. dependency arrow.
Indicative answer / marking noteC. The critical path determines the minimum completion time.
Q02 Define Section B · Short answer 1 mark
Define the term ‘milestone’ in project management.
Indicative answer / marking noteA significant point/event marking the completion of a phase or key deliverable; has zero duration.
Q03 Describe Section B · Short answer 2 marks
Describe what a dependency between two tasks means and how it is shown on a Gantt chart.
Indicative answer / marking noteDependency (1): one task cannot start (or finish) until another is complete. Shown (1): a linking arrow from predecessor to successor.
Q04 Explain Section B · Short answer 2 marks
Explain the significance of the critical path to a project manager.
Indicative answer / marking noteTasks on it have no slack; any delay delays the whole project, so the manager monitors/protects these tasks and allocates resources to them first.
Q05 Identify Section B · Short answer 2 marks
Tasks: A(2d)→C(3d); B(4d)→C. Identify the critical path and the project duration.
Indicative answer / marking noteB→C = 7 days vs A→C = 5 days. Critical path = B→C; duration = 7 days.
Q06 Outline Section B · Short answer 2 marks
Outline what ‘slack’ (float) is and why a non-critical task may have slack.
Indicative answer / marking noteSlack (1): the time a task can be delayed without delaying the project. Reason (1): it runs in parallel with a longer dependent chain, so spare time exists.
Q07 Explain Section C · Case study 3 marks
A developer falls two days behind on a task that is on the critical path. Explain the impact on the project and one way to recover.
Indicative answer / marking noteImpact (1–2): whole project end date slips by ~2 days (no slack). Recovery (1): crash the task (add resources), fast-track parallelisable work, or re-scope.
Q08 Describe Section C · Case study 2 marks
Describe two benefits of using a Gantt chart to manage a software development project.
Indicative answer / marking noteAny 2 × 1: visualises schedule/timeline; shows dependencies; identifies critical path; tracks progress vs plan; aids resource allocation/communication.
Q09 Explain Section C · Case study 3 marks
Explain how milestones and dependencies are used together to monitor whether a project is on track.
Indicative answer / marking noteDependencies (1–2): enforce task order and reveal flow-on delays. Milestones (1): checkpoints to compare planned vs actual completion, triggering corrective action if missed.
Q10 Justify Section C · Case study 3 marks
A manager wants to shorten the project. Justify why adding resources to a non-critical task will NOT help.
Indicative answer / marking noteNon-critical tasks have slack and are not the limiting chain; only shortening critical-path tasks reduces total duration. Reward correct reasoning about the critical path being the constraint.

Techniques for collecting data

Q09 Justify Section C · Case study 3 marks
Justify the use of observation in addition to interviews when analysing an existing manual process.

Q10 Discuss Section C · Case study 4 marks
Discuss the advantages and limitations of using two contrasting data-collection techniques when gathering requirements for a new app.
Indicative answer / marking notePick two (e.g. survey + interview); give 2 strengths and 2 limitations across them and note how they complement each other (breadth + depth). Reward balanced discussion.

Depicting relationships — context diagrams, data flow diagrams, use case diagrams
2026 signal: DFD/use-case core but brutal — 81% zero on DFD, 52% on includes/extends.
Q01 Identify Section A · Multiple choice 1 mark
In a data flow diagram, an open-ended rectangle (or rounded store symbol) represents a
A. process.
B. external entity.
C. data store.
D. data flow.
Indicative answer / marking noteC. A data store holds data at rest.
Q02 Define Section B · Short answer 1 mark
Define the term ‘external entity’ in a context diagram.
Indicative answer / marking noteA person, organisation or system outside the solution boundary that supplies or receives data.
Q03 Describe Section B · Short answer 2 marks
Describe the difference between a context diagram and a level-0 data flow diagram.
Indicative answer / marking noteContext (1): one process (the whole system) with external entities and net data flows. DFD level-0 (1): decomposes that into multiple processes, data stores and internal flows.
Q04 Explain Section B · Short answer 2 marks
In a use case diagram, explain the difference between an ‘include’ and an ‘extend’ relationship.
Indicative answer / marking noteInclude (1): base use case always uses the included behaviour (mandatory sub-task). Extend (1): optional/conditional behaviour added under certain conditions.
Q05 Identify Section B · Short answer 2 marks
For a login feature, identify one actor and one use case, and state their relationship.
Indicative answer / marking noteActor (1): User/Member. Use case (1): ‘Log in’. Association: the actor initiates/participates in the use case (line connecting them).
Q06 Outline Section B · Short answer 2 marks
Outline the four key symbols used in a data flow diagram.
Indicative answer / marking noteProcess; external entity; data store; data flow (arrow). Award per correct symbol/role up to 2.
Q07 Describe Section C · Case study 3 marks
A library system: members borrow and return books; staff add new titles. Describe the actors and two use cases, including one include relationship.
Indicative answer / marking noteActors (1): Member, Staff. Use cases (1): Borrow book, Return book, Add title. Include (1): ‘Borrow book’ includes ‘Verify membership’ (always required).
Q08 Explain Section C · Case study 3 marks
Explain why a context diagram is drawn before a detailed DFD, and what each contributes to analysis.
Indicative answer / marking noteContext (1–2): fixes the system boundary and external interactions first, agreeing scope. DFD (1): then details internal processing/stores once the boundary is agreed; prevents scope confusion.
Q09 Contrast Section C · Case study 3 marks
Contrast a use case diagram and a data flow diagram in terms of what each is designed to show.
Indicative answer / marking noteUse case (1–2): functional requirements — who (actors) does what (use cases). DFD (1–2): how data moves/transforms between processes and stores. Must contrast purpose.
Q10 Justify Section C · Case study 3 marks
An analyst omits data stores from a DFD. Justify why this is a significant error.
Indicative answer / marking noteData stores show where data persists; omitting them hides storage requirements, breaks data-flow balancing, and misrepresents the system, leading to design/build errors. Reward clear consequence reasoning.
U3O2·KK10 Unit 3 – Outcome 2 — Analysis & Design ▲ Very likely
Key legal requirements — privacy and copyright
2026 signal: Copyright & privacy legislation named for 2026.
Q01 Identify Section A · Multiple choice 1 mark
Which Act primarily governs how Australian organisations handle individuals' personal information?
A. Copyright Act 1968.
B. Privacy Act 1988.
C. Health Records Act 2001.
D. Spam Act 2003.
Indicative answer / marking noteB. The Privacy Act 1988 (and the Australian Privacy Principles) governs personal information handling.
Q02 State Section B · Short answer 1 mark
State what the Copyright Act 1968 protects in the context of software.
Indicative answer / marking noteThe original expression of source code and related materials as a literary work, protecting against unauthorised copying/use.
Q03 Describe Section B · Short answer 2 marks
Describe two obligations the Privacy Act 1988 places on an organisation that collects personal data.
Indicative answer / marking noteAny 2 × 1: collect only what's necessary/with consent; notify purpose; keep secure; allow access/correction; use only for stated purpose; limit disclosure.
Q04 Explain Section B · Short answer 2 marks
Explain one way a software solution can be designed to comply with the Privacy Act 1988.
Indicative answer / marking noteE.g. encrypt stored personal data + access controls; collect minimum data; obtain consent; provide data access/correction features. Reward a concrete compliance measure tied to an obligation.
Q05 Outline Section B · Short answer 2 marks
Outline the difference between privacy and security of data.
Indicative answer / marking notePrivacy (1): who may access/use personal data and for what purpose (rights). Security (1): technical/physical protection of data from unauthorised access/loss.
Q06 Identify Section B · Short answer 2 marks
A developer copies a function from a tutorial website into a commercial product. Identify the legal issue and one way to address it.
Indicative answer / marking noteIssue (1): potential copyright infringement of the author's code. Address (1): check/obtain a licence (e.g. permissive open-source) or write own implementation/attribute as required.
